# 版本命名规范

以后每个新版本用这个格式：

| 字段 | 含义 | 示例 |
|------|------|------|
| **project** | 仓库/文件夹 ID（英文，部署用） | `hijuly-diary` |
| **name** | 对外品牌名（中文） | `小螃蟹看世界` |
| **version** | 语义化版本 `主.次.修` | `4.0.0` |

显示格式：**`project | name | v4.0.0`**

例：`hijuly-diary | 小螃蟹看世界 | v4.0.0`

- **主版本**：大改版（整站 rebrand、信息架构变了）
- **次版本**：新功能区块（宠物蟹、地图筛选等）
- **修版本**：修 bug、换图、改链接

改版本时编辑 `data/version.json`，页脚会自动显示。
